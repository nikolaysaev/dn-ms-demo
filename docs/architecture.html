4 · Retrieval — hybrid pool

dense legembed query (OpenAI) → Qdrant top-30 → note-filter (perfume only)
＋
lexical legcached name map (10-min TTL) → BG-suffix stems, IDF-weighted → top-12 by head-noun match
→
rerankbase + name bonus + translit bonus + _lex_score×0.12 + stock/gender penalties
→
top-kbest + alternatives, family-aware grouping
Why: dense matches context and misses the decisive noun — „входящи маркучи" returned „Изходящ маркуч"; „пружини за барабан" returned „преграда за барабан". Only the lexical leg knows which candidate carried the head noun; its IDF score rides into rerank as the tie-breaker. Fixed 9/11 browse failures; the rest need category names (client data).

5 · Identity guards self-validating — the data decides, not vocabularies

Match-count guard exact_code.py

A term identifying > 8 products is a spec fragment („80n", „11mm", „260"), not a code — dropped. One SQL GROUP BY answers all terms; retrieve-by-id kills the old scroll id-order bias.

Distinct-appliance guard fitment.py

Fitment terms whose matches span > 8 distinct appliances are not identifiers → ignored. Terms and substring evidence need ≥ 5 normalized chars (junk rows with appliance_code "80N" become unmatchable).

Digit-tail ≥ 5 wak260 bug class

Model digit-tails („wak260"→„260") only count with ≥ 5 chars — bare „260" matched a vacuum pump V-i260 and a gasket in prod.

Router-model rescue ask.py

Router can mislabel a part code as appliance model („V-i260"). The identity index adjudicates: a full "model" token identifying 1–8 products IS a part code — kept. Digit fragments („885") never eligible.

6 · Sticker-photo flow the headline feature

  1. Widget upload POST /v1/chat/upload (multipart)Image saved per-session, persisted as user message.
  2. Vision extraction POST /api/vision/sticker (parts stores only)gpt-4o-mini vision, prompt forbids guessing. Output: brand, model, full_model, serial, PNC + readable flag.
  3. Readable → normal ask turnSynthesized Bulgarian message („Уредът е с марка BOSCH, модел WAE16422IT/01…") through the standard pipeline → „Намерих 22 съвместими части. Каква част търсите?" + part-type chips.
  4. Unreadable → human reviewOriginal behavior kept: escalation, admin notification, „специалист ще я прегледа".
