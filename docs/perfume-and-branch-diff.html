Part I · The perfume vertical — all cases

A perfume store is vertical: "perfume" with a slots profile (notes, gender, occasion, budget) and no parts capabilities — so store_is_parts=False, agent_loop is off, and every request runs the classic cascade. The engine ships no copy: all customer-facing text comes from the store's own AssistantProfile (intent_messages/intent_chips).

Conversational cases (cascade order)

  1. Welcome — empty first turn → the store's authored welcome + starter chips; dashboard appends active promotions.
  2. Canned intent gates — greeting, self-intro, off-topic, thanks/goodbye, bare „не" (context-aware: after greeting vs after results), delivery, returns/complaints, payment/order-status, contact/„искам оператор" (sets human_takeover). Each answers from profile copy and exits.
  3. PATH A — „Покажи препоръки" button → starts the guided profiling collection.
  4. PATH E — active collection session — the multi-turn preference Q&A (notes → gender → occasion → budget); runs before B/C/D so answers („дамски", „до 50 лв") aren't re-classified as fresh queries.
  5. PATH C — inline preference — „харесвам Baccarat Rouge и Yara" → _profiling_oneshot: the named perfumes are profiled by LLM into notes/character and matched in one shot.
  6. Router + memory merge — gpt-class router extracts gender/notes/budget into session memory; „show more" reuses the last result set; price questions answer from the last shown product.
  7. Semantic search + note filter — dense retrieval, then _apply_perfume_note_filter: notes requested with „с …" („с ванилия и уд") become required; under STRICT_NOTE_MATCH an empty result retries exact-note Qdrant search. The Bulgarian preposition „с" is why this filter is perfume-only — on parts it read „маркуч с аквастоп" as a scent note.
  8. Gender handling — Qdrant payload filter on canonical gender + a rerank backstop that demotes clear opposite-gender matches written before normalization.
  9. Cross-script matching — „латафа яра" ≈ Lattafa Yara: transliteration bonus + tightness bonus (exact-length name beats „Yara Moi").
  10. Post-result — perfume refine chips (__refine: payloads: по-евтино/по-луксозно/друг характер), Fragella enrichment (notes/imagery via per-store key), grounded reasoning card with perfume trait vocabulary.

Architecture: main → branch

main (before)

One 3,350-line handler cascade in ask.py — 43 exit points, correctness = implicit ordering. Identity lookups via Qdrant scroll (id-order truncation bias, per-term count calls). Fitment capped at 80 rows/product (94% of data discarded). Photos → human review only. Dense-only retrieval. No external agent surfaces.

branch (after)

Deterministic pre-layer → tool-calling agent (per-store flag, cascade as automatic fallback) → server-side composition. Identity in SQL (uncapped fitment), payloads fetched by id. Vision intake, hybrid dense+lexical retrieval with a relevance floor, session-memory persistence for photo readings, MCP + product-feed surfaces, analytics KPIs.
